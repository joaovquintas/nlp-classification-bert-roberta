Descrição de cada diretório e arquivo:
data/: Contém o dataset original ou qualquer outro dado necessário. Esse diretório pode incluir também arquivos de pré-processamento ou resultados intermediários.

models/: Contém todos os modelos definidos, como o BERT e RoBERTa. O código que define o modelo fica aqui. Você pode criar mais arquivos de modelo, como bert_classifier.py, roberta_classifier.py, etc.

bert_classifier.py: Define o modelo e os componentes para o BERT-base-uncased.
roberta_classifier.py: Define o modelo e os componentes para o RoBERTa-base.
utils.py: Contém funções auxiliares, como cálculo de métricas (precisão, F1, etc.), e funções de carga de dados, pré-processamento, etc.
training/: Aqui fica todo o código relacionado ao treinamento do modelo e à avaliação. Você pode separar a lógica de treinamento e as métricas para que seja mais fácil de modificar.

train.py: Responsável por treinar os modelos, definir as funções de treinamento e avaliação, carregar os dados, e salvar os resultados em arquivos.
metrics.py: Contém funções para calcular as métricas (como precisão, F1, etc.) de forma reutilizável para ambos os modelos.
outputs/: Esse diretório armazenará os resultados dos testes e métricas. Cada modelo pode ter um arquivo específico onde você pode salvar a acurácia, F1-score e outros resultados.

requirements.txt: Contém todas as dependências do projeto. Você pode gerar esse arquivo com pip freeze > requirements.txt para garantir que todas as bibliotecas necessárias sejam instaladas.

README.md: Um arquivo com a descrição do projeto, como usá-lo, e quaisquer instruções de configuração.

main.py: Arquivo principal que roda o projeto. Nele, você importa os módulos, configura o modelo, inicia o treinamento e avalia o modelo.project/
